# Query data

After connecting to the blockchain via an `LCDClient` instance, you can begin querying data. Data access is organized into various module APIs, which are accessible from within the `LCDClient` instance. Because they make HTTP requests in the background, they are Promises that can be awaited in order to not block during network IO.

Each module has its own set of querying functions. To get a comprehensive list, explore the module documentation:

- [`auth`](https://featherjs.pages.dev/classes/AuthAPI.html)
- [`bank`](https://featherjs.pages.dev/classes/BankAPI.html)
- [`distribution`](https://featherjs.pages.dev/classes/DistributionAPI.html)
- [`gov`](https://featherjs.pages.dev/classes/GovAPI.html)
- [`mint`](https://featherjs.pages.dev/classes/MintAPI.html)
- [`msgauth`](https://featherjs.pages.dev/classes/MsgAuthAPI.html)
- [`slashing`](https://featherjs.pages.dev/classes/SlashingAPI.html)
- [`staking`](https://featherjs.pages.dev/classes/StakingAPI.html)
- [`supply`](https://featherjs.pages.dev/classes/SupplyAPI.html)
- [`tendermint`](https://featherjs.pages.dev/classes/TendermintAPI.html)
- [`tx`](https://featherjs.pages.dev/classes/TxAPI.html)
- [`wasm`](https://featherjs.pages.dev/classes/WasmAPI.html)
